Config files control the behavior of ScChrom. They usual have the filename extension .sccf, but every file that contains utf-8 encoded text will work. The build in editor provides more information about the available parameters and methods provided by the JsController. Make sure to check out the config file examples.

Parameter

Every parameter starts on a new line with -- followed by its name, a = and the parameters value, like: 
  --parameter=value
The value can span over multiple lines. It will end once a line with a new parameter occurs or the config file ends.
Some parameters support stacking. This provides a way to use the same parameter for different contexts. Every stacked value is identified by their respective stack key, like: 
  --parameter<‍stackKey1>=value1
  --parameter<‍stackKey2>=value2
In the editor all available parameters with a description, default values and other information can be viewed.

JsController

Some parameters, especially the eventhandler, allow JavaScript injection into the browsers or Jint context. Additional functionallity outside of these two contexts is available via methods of the JsControllers.
In the Jint context a method is called via the JsControllers and the specific method name like 
  JsController.methodName();
  var value1 = JsController1.methodName1();
JsController methods inside the Jint context block until they are done or return a value.

JsController methods inside the browsers context are called via the injected ScChrom object and are executed asynchronously as a promise like: 
  // No blocking here
  ScChrom.JsController.methodName();
  // if the return value is needed await can be used
  var value1 = await ScChrom.JsController1.methodName1();
